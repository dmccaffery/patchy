patchy {{ .Chart.AppVersion }} deployed to namespace {{ .Release.Namespace }}
(agent sandbox: {{ .Values.agent.namespace }}).

The CRs are the state machine: findings, investigations, and remediations
live as patchy.bitwisemedia.uk custom resources in {{ .Release.Namespace }}
(kubectl get patchy -n {{ .Release.Namespace }}).

The pipeline idles until two things exist:

  1. An Integration and a Forge custom resource (render them with the
     `integrations:` / `forges:` values arrays, or apply your own). The
     Integration validates webhooks and projects tracking issues; the Forge
     clones and pushes repositories.
  2. The Secrets they reference, created out of band:
       * the forge/provider credential (keys: appID + privateKey, or token,
         plus webhookSecret on Integration secrets) in namespace
         {{ .Release.Namespace }}, named by each CR's spec.secretRef.
       * a model credential per enabled harness in namespace
         {{ .Values.agent.namespace }} — one Secret for each enabled
         agent.runners.<harness> (default:
         {{- range $id, $r := .Values.agent.runners }}{{ if $r.enabled }} {{ $r.secret }} (key {{ $r.secretKey }}, as {{ $r.secretEnv }}){{ end }}{{- end }}).
         A harness is enabled only when its runner is enabled AND its
         credential exists; every allowlisted model needs one enabled harness
         that supports it, or the controllers refuse to start.
     See deploy/kustomize/base/secrets.example.yaml in the patchy repository
     for the exact shapes and kubectl one-liners.

For defence in depth, one of:

  * GKE Dataplane V2 — recreate or update the cluster with
    --enable-fqdn-network-policy, then re-reconcile; mode auto picks it up.
  * Cilium — needs DNS-based policy (the DNS proxy); auto picks it up.
  * Istio — set mode: istio (never auto-selected: it needs native sidecars
    and namespace injection, which CRD presence does not imply).

Note that auto can only see the cluster when Helm renders against one; an
off-cluster `helm template` always reports none.
