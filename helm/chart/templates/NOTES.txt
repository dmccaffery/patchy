patchy {{ .Chart.AppVersion }} deployed to namespace {{ .Release.Namespace }}
(agent sandbox: {{ .Values.agent.namespace }}).

Nothing works until the three out-of-band Secrets exist:

  1. {{ .Values.github.appSecret }} (namespace {{ .Release.Namespace }})
     keys: app-id, private-key.pem — the GitHub App identity.
  2. {{ .Values.github.webhookSecret }} (namespace {{ .Release.Namespace }})
     key: secret — the webhook HMAC secret configured on the App.
  3. {{ .Values.anthropic.secret }} (namespace {{ .Values.agent.namespace }})
     key: {{ .Values.anthropic.secretKey }} — the model credential, injected as
     {{ .Values.anthropic.secretEnv }}. Without it every agent Job pod fails
     with CreateContainerConfigError.

See deploy/kustomize/base/secrets.example.yaml in the patchy repository for
the exact shapes and kubectl one-liners.

{{- if or .Values.webhook.ingress.enabled .Values.webhook.httpRoute.enabled }}
Set the GitHub App's (single) webhook URL to:

  https://{{ .Values.webhook.host }}/webhook

The webhook-controller behind it validates each delivery and routes it to
the controllers that consume its event type; it holds no GitHub credential.
{{- else }}
No webhook entry point is enabled. Expose the webhook-controller (it validates
deliveries and routes them to the controllers, holding no GitHub credential)
with webhook.host plus webhook.ingress.enabled or webhook.httpRoute.enabled —
or bring your own routing to the {{ include "patchy.fullname" . }}-webhook-controller
Service — then point the GitHub App's single webhook URL at
https://<host>/webhook.
{{- end }}
{{- if not (or .Values.agent.networkPolicy.cilium.enabled .Values.agent.networkPolicy.istio.enabled) }}

NOTE: the agent sandbox currently has no hostname-level egress policy — the
base NetworkPolicy allows TCP 443 to anywhere, and credential absence is the
control. If your cluster runs Cilium or Istio, enable
agent.networkPolicy.cilium.enabled or agent.networkPolicy.istio.enabled for
defence in depth.
{{- end }}

labels the agent namespace it creates):

  kubectl label ns {{ .Release.Namespace }} \
    pod-security.kubernetes.io/enforce=restricted \
    pod-security.kubernetes.io/audit=restricted \
    pod-security.kubernetes.io/warn=restricted
