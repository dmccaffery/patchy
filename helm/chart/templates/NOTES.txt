patchy {{ .Chart.AppVersion }} deployed to namespace {{ .Release.Namespace }}
(agent sandbox: {{ .Values.agent.namespace }}).

Nothing works until the three out-of-band Secrets exist:

  1. {{ .Values.github.appSecret }} (namespace {{ .Release.Namespace }})
     keys: app-id, private-key.pem — the GitHub App identity.
  2. {{ .Values.github.webhookSecret }} (namespace {{ .Release.Namespace }})
     key: secret — the webhook HMAC secret configured on the App.
  3. {{ .Values.anthropic.secret }} (namespace {{ .Values.agent.namespace }})
     key: {{ .Values.anthropic.secretKey }} — the model API key. Without it every
     agent Job pod fails with CreateContainerConfigError.

See deploy/kustomize/base/secrets.example.yaml in the patchy repository for
the exact shapes and kubectl one-liners.

Each controller needs its own webhook URL on the GitHub App, all signed with
the same secret:

  POST http://{{ include "patchy.fullname" . }}-source-controller:{{ .Values.sourceController.service.port }}/webhook       (code_scanning_alert)
  POST http://{{ include "patchy.fullname" . }}-context-controller:{{ .Values.contextController.service.port }}/webhook      (issues)
  POST http://{{ include "patchy.fullname" . }}-remediation-controller:{{ .Values.remediationController.service.port }}/webhook  (issues)

Expose them per controller with <controller>.ingress or <controller>.httpRoute
(scoped to /webhook), or with your own Gateway.

    pod-security.kubernetes.io/enforce=restricted \
